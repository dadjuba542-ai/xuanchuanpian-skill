# 宣传片 Seedance 生产流水线

## 前置依赖

- `seedance` SKILL（加载方式：`skill_view(name='seedance')`）
- 相机四维编码系统、深度美学约束、剪辑节奏公式 均参考 `seedance` 的 references

---

## 工作流总览

```
Phase 1 ──→ Phase 2 ──→ Phase 3 ──→ Phase 4 ──→ Phase 5 ──→ Phase 6
文案拆画面    N宫格图片     生图（外     图片→AI     人工审改     N张图+1条
  类型         提示词        部工具）     生成视频                 提示词→
                                          提示词                  1条15s视频
```

| 阶段 | 自动化 | 人工介入 | 交付物 |
|------|--------|---------|--------|
| Phase 1 | 子 agent 出候选 | 勾选确认 | 文案→画面类型对照表 |
| Phase 2 | 子 agent 按模板生成 | 确认宫格数 + 微调 | N 条图片提示词（用于生图） |
| Phase 3 | ❌ 外部工具 | 手动生图 | N 张图片 |

| Phase 5 | — | 全量审改 | 最终视频提示词定稿 |
| Phase 6 | — | 手动上传 N 图+1 条提示词到即梦 | 1 条 15s 视频成片 |

## Phase 1：文案拆画面类型

**角色**：子 agent（主会话可审） + 人工勾选确认
**输入**：宣传片文案（逐句或逐段）
**输出**：文案句 → 画面类型 对照表

### Step 1：子 agent 分析

对每一句文案，AI 自动出 2-3 个画面类型候选：

| 画面类型 | 适用场景 | 示例 |
|---------|---------|------|
| **叙事型** | 讲品牌故事/用户故事 | 人物在真实场景中做事 |
| **产品展示型** | 产品特写/功能演示 | 产品 360° 环绕展示 |
| **情绪氛围型** | 烘托情感/建立品牌调性 | 光影/材质/氛围空镜 |
| **转场型** | 场景过渡/节奏变化 | 推拉摇移/匹配剪辑 |
| **特写型** | 强调细节/质感 | 局部放大/微距 |
| **空镜型** | 建立环境/交代背景 | 城市/自然/室内全景 |
| **标题型** | 关键信息/字幕画面 | 留白够加文字的构图 |

### Step 2：人工确认

- 从候选中选择最合适的类型
- 可调整或自定义
- **交付物**：文案句 × 画面类型 对照表（如 3 句文案 → 可能分配 4-6 个画面类型，一句文案可拆成多个画面）

## Phase 2：画面类型→N宫格图片提示词

**角色**：子 agent 生成 → 人工确认宫格数和内容 → 微调
**输入**：Phase 1 的对照表
**输出**：N 条图片提示词（用于 Phase 3 生图）

### Step 1：AI 建议宫格数 + 人工判断

宫格数决定了分镜的精细度和视频的节奏稳定性，类似漫画分镜——格数越多，每格承载的信息越少，画面切换越稳。AI 根据画面类型数量和分镜复杂度给出建议：

| 宫格数 | 分镜精细度 | 节奏效果 | 适合场景 |
|--------|-----------|---------|---------|
| **6 宫格** | 粗略拆分，每格承载较多画面信息 | 单格停留长，节奏偏慢，适合信息密度大、节奏舒缓的宣传片 | 画面类型 ≤6，镜头数量少，产品展示/品牌形象 |
| **9 宫格** | 中等拆分，每格一个明确的画面概念 | 节奏适中，每格 1.5-2s，信息节奏稳定 | 画面类型 7-10，标准宣传片，叙事+产品混合 |
| **12 宫格** | 精细拆分，每格一个单一镜头/动作 | 节奏紧凑，每格 1-1.5s，画面变化快，稳定性高 | 画面类型 ≥11，需要精细转场、快节奏切换 |

> **核心逻辑**：分镜越精细（宫格越多）→ 每格画面信息越少 → 视频节奏越稳定，越不容易出现"一段画面空洞"或"运镜不听使唤"。类似于漫画里格越小、动作拆得越细，读起来越流畅。
>
> 每个宫格 = 一张图片（用于视频的参考画面）。
> N 宫格 = N 张图，最终搭配 1 条视频提示词一起喂给即梦，生成 1 条 15s 视频。
> 宫格数越多，单格承载内容越少、分镜越精细，但生图工作量越大。

AI 给出推荐 + 理由。**最终由人工决定**。

### Step 2：生成每条图片提示词（中英双语）

每条图片提示词用于在生图平台（NanoBanana / 即梦图片 / MJ）生成一张宫格参考图。
采用 **中文（上）+ 英文（下）** 结构——中文供人工参考确认，英文复制到生图工具。

使用 `seedance` SKILL 中「图片生成提示词（角色参考图 & 首帧图）」的格式。

#### 图片提示词格式（中英双语）

**中文（参考用）**：
```
[画幅比]。[风格锚定]，[构图+主体位置]。[主体动作/状态+关键特征]。[环境/背景细节]。[光影层次]。[色调氛围]。[画质约束]。禁止：任何文字、字幕、LOGO、水印
```

**英文（生图用）**：
```
[aspect ratio]. [style anchor], [composition + subject position]. [subject action/state + key features]. [environment/background details]. [lighting layers]. [color tone]. [quality constraints]. No text, subtitles, logos or watermarks
```

> 中文写清楚细节描述（供人看），英文简洁准确（给模型用）。
> 英文不需要逐字翻译中文，优先用模型理解最准的英文关键词。

#### 各画面类型适配

| 画面类型 | 图片提示词要点 |
|---------|--------------|
| **叙事型** | 有人物、有场景、有动态感（人物在做事/互动） |
| **产品展示型** | 产品居中或黄金分割，质感清晰，背景干净 |
| **情绪氛围型** | 光影氛围为主，构图留白多，可无明确主体 |
| **转场型** | 画面元素简洁，有明显视觉动势/方向性 |
| **特写型** | 极致细节，背景虚化，微观质感 |
| **空镜型** | 大场景/环境为主，构图开阔，色调统一 |
| **标题型** | 构图留白≥50%（给后期文字留空间），简洁 |

#### 画幅比约束

全部宫格**画幅比必须统一**：
- 竖版（9:16）：抖音/视频号/小红书 → 默认
- 横版（16:9）：官网/发布会/大屏展示

> 生图画幅比 = 视频目标画幅比。要 9:16 视频就生 9:16 图，否则即梦裁切会切掉关键构图。

### Step 3：输出格式（中英双语）

```
## 宣传片 N 宫格图片提示词（X宫格）

**画幅比**：[16:9 / 9:16]
**宫格布局建议**：[如 3×2 / 3×3 / 4×3]
**生图平台建议**：[NanoBanana / 即梦图片生成 / MJ]

### 宫格 1：[画面类型]

**中文（参考）**：
[完整图片提示词·中文]

**EN**：
[complete image prompt in English]

### 宫格 2：[画面类型]

**中文（参考）**：
[完整图片提示词·中文]

**EN**：
[complete image prompt in English]

...
```

### Step 4：人工微调

- 确认每条图片提示词对画面类型的匹配度
- 确认画幅比统一
- **交付物**：N 条图片提示词（锁定，可直接复制到生图工具）

## Phase 3：提示词→生图（人工·外部工具）

**角色**：纯人工执行
**输入**：Phase 2 的 N 条提示词
**工具**：推荐平台（以下任选）
- **NanoBanana**（默认，效果好）
- **即梦图片生成**
- **Midjourney / DALL·E**（英文提示词需自行转换）

### 操作指引

1. 每条提示词复制到生图工具
2. 画幅比与 Phase 2 一致
3. 生成后保存到统一文件夹，**按 `grid_01.png ~ grid_N.png` 命名**
4. **交付物**：N 张图片本地保存

---

## Phase 4：图片 → AI 生成 1 条视频提示词

**角色**：子 agent + 多模态模型
**输入**：N 张图片（来自 Phase 3）
**输出**：**1 条** Seedance 视频提示词（引用 @图片1~@图片N，生成 1 条 15s 视频）

> **核心逻辑**：N 张宫格图不是分别生 N 条视频，而是作为参考素材一起喂给**1 条**视频提示词。
> 提示词用 `@图片1`~`@图片N` 引用各宫格，按时间戳分镜法安排画面切换顺序。

### Step 1：分析 N 张图片

用 `mcp_minimax_understand_image` 或 `vision_analyze` 逐张分析每张宫格图：

- 主体/构图/光影/色调
- 画面类型与文案的对应关系
- 哪些元素是独特的（需要保留），哪些是通用的（可以做过渡/衔接）

同时判断：
- **宫格排列顺序**：文案的叙事逻辑决定了宫格在视频中的出现顺序
- **过渡方式**：相邻宫格之间是硬切还是有匹配点（颜色/形状/动势）

### Step 2：生成 1 条视频提示词（中英双语）

采用 **中文（上）+ 英文（下）** 结构——中文供人工参考确认，英文复制到即梦。

使用 `seedance` SKILL 的视频提示词模板，**按画面类型选择**：

| 画面类型 | 推荐模板 |
|---------|---------|
| 以叙事/情绪为主的宣传片 | **史诗/大制作结构**（品质锚定 + 大气连贯 + 光影三层） |
| 以产品展示为主的宣传片 | **时间戳分镜法**（13-15s 精确控制每段时间） |

#### 模板：时间戳分镜法（推荐，精确控制每格画面）

**中文（参考用）**：
```
[15秒][品质锚定：渲染引擎+画质规格]。[核心氛围宣言]。
@图片1~@图片N 作为画面参考，按以下时间戳依次展现：

0-3秒：[@图片1 中的画面 + 运镜 + 音效]；
4-8秒：[@图片2 中的画面 + 运镜 + 音效]；
9-12秒：[@图片3 中的画面 + 运镜 + 音效]；
13-15秒：[@图片N 中的画面 + 运镜 + 音效]。

光影：[光源] + [光行为] + [色调]。
禁止：任何文字、字幕、LOGO 或水印
```

**英文（即梦用）**：
```
[15s][quality anchor: render engine + spec]. [core atmosphere statement].
@image1~@imageN as visual reference, unfolding in sequence:

0-3s: [scene from @image1 + camera movement + sound effect];
4-8s: [scene from @image2 + camera movement + sound effect];
9-12s: [scene from @image3 + camera movement + sound effect];
13-15s: [scene from @imageN + camera movement + sound effect].

Lighting: [light source] + [light behavior] + [color tone].
No text, subtitles, logos or watermarks
```

> 英文模板的关键词参考 `seedance` 的相机四维编码（Z/Y/X/F）和美学约束词汇。
> 如果即梦（中文平台）对英文支持不好，可改回中文直接贴，但优先用英文。

#### 模板（进阶）：史诗/大制作结构

**中文（参考用）**：
```
15秒[品质锚定：UnrealEngine5渲染，工业光魔级VFX]。[整体氛围宣言]。
[大气连贯声明：全片统一的物理效果]。
@图片1~@图片N 作为画面参考。

0-Xs：[@图片1/@图片2 组合画面 + 运镜 + 帧率]；
...（时间戳分镜，每段引用对应 @图片N）...

光影：[①光源] + [②光行为] + [③色调]。
[收束句 + 张力宣言]。
禁止：任何文字、字幕、LOGO 或水印
```

**英文（即梦用）**：
```
15s [quality anchor: Unreal Engine 5, ILM-grade VFX]. [overall atmosphere statement].
[global continuity declaration: unified physical effects throughout].
@image1~@imageN as visual reference.

0-Xs: [combined scene from @image1/@image2 + camera movement + frame rate];
...

Lighting: [light source] + [light behavior] + [color tone].
[closing statement + tension manifesto].
No text, subtitles, logos or watermarks
```

#### 运镜控制（来自 `seedance` 相机四维编码）

每条时间戳段中嵌入运镜定义：

```
一个完整镜头 = [Z 距离] + [Y 高度] + [X 方位] + [F 滤镜] + [运动] + [节奏] + [约束]
```

- 铁律：每条时间戳段最多双轴运动
- 默认：Z4 中近景 + Y4 平视 + X2 四分之三侧 + 50mm + 浅景深 + 缓入缓出

### Step 3：输出格式（中英双语）

```

**引用素材**：@图片1~@图片X（共 X 张）
**宫格顺序**：N → ...
**各格过渡方式**：[硬切 / 匹配过渡 / 统一运镜连贯]
**画幅比**：[9:16 / 16:9]

### 视频提示词

**中文（参考）**：
[完整中文提示词]

**EN（即梦用）**：
[complete English prompt, paste directly into dreamina]

### 设计说明

| 宫格 | 画面类型 | 出场时间 | 运镜方式 |
|------|---------|---------|---------|
| @图片1 | [类型] | 0-3s | [Z/Y/X/F 编码 + 运动] |
| @图片2 | [类型] | 4-7s | ... |
| ... | ... | ... | ... |

```

- 标注字符数（`len()` 实测，≤2000）
- **交付物**：1 条 D 级视频提示词（可直接复制到即梦）
- 完整的深度美学约束（渲染标准+冷暖色调+极简构图）参考 `seedance` SKILL

## Phase 5：人工审改

**角色**：纯人工
**输入**：Phase 4 的 **1 条**视频提示词 + N 张图片

### 检查清单

- [ ] 提示词中 `@图片1`~`@图片N` 的引用编号与上传顺序一致
- [ ] 每条时间戳段的画面描述与对应的宫格图匹配
- [ ] 运镜合理：不从图片做不到的角度开始
- [ ] 全片色调/风格整体统一（而非逐格割裂）
- [ ] 无写实真人面部素材（即梦拦截）
- [ ] 提示词 ≤2000 字符（`len()` 实测）
- [ ] 有禁止文字/LOGO/水印的声明
- [ ] 画幅比统一
- [ ] 各宫格之间的过渡方式明确（硬切 / 匹配过渡 / 统一运镜连贯）

**交付物**：1 条最终视频提示词定稿

## Phase 6：N 张图 + 1 条提示词 → 1 条 15s 视频（人工操作）

### 操作指引

1. 打开即梦平台（dreamina）
2. 上传 N 张宫格图片（`@图片1`~`@图片N`）
3. 粘贴 Phase 5 的 **1 条**视频提示词
4. 选择模型：`seedance2.0_vip`
5. 分辨率：720p（试片）→ 效果满意后升 1080p
6. 生成 1 条 15s 视频
7. 用剪映加音效、加文字（标题/字幕/LOGO 均在剪映加）

### 建议流程

- **先试片**：确认第一版效果（宫格切换顺序对不对、风格统一不统一）
- **不满意 → 回 Phase 5 改提示词**，不要重复跑同一版
- **后期加文字**：所有标题/字幕/LOGO 在剪映加，避免视频内文字错乱

## 🧠 翻车预防 & 实战提示

### 提示词文字在视频中会错乱

Seedance 2.0 会把提示词中描述的文字/标语/字幕当成画面元素渲染，结果通常是乱码/错字/变形。**解药**：提示词中不要包含任何文字描述（标题、标语、字幕等），所有文字/LOGO 在后期用剪映添加。

### 先试片再定稿

1 条视频提示词覆盖 N 张宫格图，第一次跑可能宫格切换顺序不对、运镜生硬。**先跑一版看看效果**，不满意回 Phase 5 改提示词，不用重复生图。

### 宫格图之间的过渡

- **提示词中写清顺序**：用时间戳分镜法明确 `0-3秒：@图片1画面` → `4-8秒：@图片2画面`...
- **即梦会自动过渡**，但效果取决于提示词写得清不清晰
- 如果相邻宫格差异大（如空镜→特写），加一句过渡描述

### 宫格图风格统一（最重要）

N 张图如果画风不一致（色调/光影/构图），即梦很难在 1 条视频里弥合。
**解法**：
1. 生图时就用统一的风格锚定（参考 `seedance` 美学约束）
2. 如果生图平台风格不统一，在 Phase 4 视频提示词开头加全局风格锁定
3. 后期剪映加统一色调滤镜兜底

### 生图阶段画幅比

生图时画幅比必须与视频目标一致（要 9:16 视频 → 生 9:16 图；要 16:9 → 生 16:9 图），否则图到即梦要裁切，关键构图被切掉。

### 关于文字生成能力的模型演进

当前（2026-07）Seedance 2.0 视频内文字不可靠。但模型在持续进化——保持关注 Seedance 3.x 或后续版本的更新，如果原生视频文字生成能力成熟了，Phase 6 的"剪映加文字"建议可以升级为"提示词内加文字"。**SKILL 中的提示词本身仍然按"不含文字"写，未来版本更新时修改此提示即可。**
